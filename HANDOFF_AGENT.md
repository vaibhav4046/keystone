## 1. What Keystone is

A **merge-safety engine on a code call graph**. Problem ("the 2am break"): two merge requests pass
review, touch different files, have **no Git conflict**, and still break production together —
because both change functions the SAME downstream code depends on. Git, review, CODEOWNERS can't
see it. Keystone reads the call graph, computes each change's **blast radius** (dependents); when
two changes' blast radii overlap = a **silent collision**. It surfaces it before merge, **vetoes a
merge two humans already approved**, and locks the decision in a tamper-evident ledger.

Capabilities shipped: silent-collision detection, **multi-collision engine** (finds ALL N
collisions + a **safe merge order**), **single-point-of-failure** %, blast-radius cockpit,
**Engineering Harness** (gates AI-agent/bot MRs -> ALLOW/HOLD/BLOCK), hash-chained **audit ledger**.
